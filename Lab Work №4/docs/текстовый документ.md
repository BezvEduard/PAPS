
# Лабораторная работа №4
**Тема:** Проектирование REST API
**Цель работы:** Получить опыт проектирования программного интерфейса.

## 1. Документация по API

В рамках лабораторной работы проектируется REST API для серверной части системы приоритизации импортозамещения. API предназначен для работы с загруженными наборами данных, запуском анализа, управлением профилями весов и получением результатов ранжирования.

### 1.1 Принятые проектные решения

Ниже зафиксированы ключевые проектные решения при проектировании API.

1. **Используется REST-ориентированный стиль.**
   Все операции оформлены как работа с ресурсами (`datasets`, `analyses`, `weight-profiles`, `results`), а не как RPC-вызовы.

2. **Используется версионирование через префикс URL.**
   Все endpoint-ы начинаются с `/api/v1/...`. Это позволит в будущем развивать API без поломки старых клиентов.

3. **Для именования используются существительные во множественном числе.**
   Например: `/datasets`, `/analyses`, `/weight-profiles`. Это делает структуру URI единообразной и предсказуемой.

4. **Для передачи данных используется формат JSON.**
   Запросы с телом и ответы возвращаются в `application/json; charset=utf-8`.

5. **Используются стандартные HTTP-методы.**
   `GET` — получение данных, `POST` — создание ресурса, `PUT` — обновление ресурса, `DELETE` — удаление ресурса.

6. **Используются стандартные коды ответа HTTP.**
   `200 OK`, `201 Created`, `204 No Content`, `400 Bad Request`, `404 Not Found`, `422 Unprocessable Entity`.

7. **В ответах используются идентификаторы ресурсов и машиночитаемые поля.**
   Каждый создаваемый объект получает поле `id`, а ответы содержат структуры, пригодные для дальнейшей автоматической обработки.

8. **Для связанных ресурсов используются вложенные маршруты.**
   Например: `/analyses/{analysis_id}/results` — получение результатов конкретного анализа.

9. **Дата и время передаются в ISO 8601.**
   Это упрощает обмен данными между клиентом и сервером и делает формат однозначным.

10. **Обновление профиля весов выполняется целиком через PUT.**
    Клиент передаёт полное новое состояние профиля весов, что упрощает реализацию и тестирование.

11. **Удаление набора данных через DELETE считается допустимым только в учебной реализации.**
    В промышленной системе можно было бы использовать мягкое удаление, но в рамках лабораторной выбран более простой и понятный вариант.

12. **Учебная реализация не включает полноценную аутентификацию.**
    Это решение соответствует принципу YAGNI для лабораторной работы: сначала проектируется и реализуется API предметной области.

### 1.2 Общие сведения

- **Базовый URL:** `http://localhost:8000/api/v1`
- **Формат запросов/ответов:** `application/json`
- **Кодировка:** `UTF-8`

Общие заголовки для запросов с телом:
```http
Content-Type: application/json
Accept: application/json
```

## 1.3 Описание endpoint-ов

### 1) Получение списка наборов данных

- **Метод:** `GET`
- **Endpoint:** `/datasets`

**Описание:**
Возвращает список всех загруженных наборов данных.

**Параметры запроса:**
Отсутствуют.

**Пример запроса:**
```http
GET /api/v1/datasets HTTP/1.1
Host: localhost:8000
Accept: application/json
```

**Пример ответа `200 OK`:**
```json
[
  {
    "id": 1,
    "file_name": "trade_2024.xlsx",
    "source_format": "xlsx",
    "records_count": 1250,
    "uploaded_at": "2026-03-18T10:00:00",
    "status": "uploaded"
  }
]
```

---

### 2) Создание набора данных

- **Метод:** `POST`
- **Endpoint:** `/datasets`

**Описание:**
Создаёт запись о новом наборе данных, загруженном в систему.

**Тело запроса:**
```json
{
  "file_name": "trade_2025.csv",
  "source_format": "csv",
  "records_count": 2400,
  "status": "uploaded"
}
```

**Поля тела запроса:**
- `file_name` — имя файла;
- `source_format` — формат файла (`csv` или `xlsx`);
- `records_count` — количество записей;
- `status` — статус загрузки.

**Пример ответа `201 Created`:**
```json
{
  "id": 2,
  "file_name": "trade_2025.csv",
  "source_format": "csv",
  "records_count": 2400,
  "uploaded_at": "2026-03-18T12:10:00",
  "status": "uploaded"
}
```

---

### 3) Получение набора данных по идентификатору

- **Метод:** `GET`
- **Endpoint:** `/datasets/{dataset_id}`

**Описание:**
Возвращает сведения о конкретном наборе данных.

**Параметры пути:**
- `dataset_id` — идентификатор набора данных.

**Пример ответа `200 OK`:**
```json
{
  "id": 1,
  "file_name": "trade_2024.xlsx",
  "source_format": "xlsx",
  "records_count": 1250,
  "uploaded_at": "2026-03-18T10:00:00",
  "status": "uploaded"
}
```

**Пример ответа `404 Not Found`:**
```json
{
  "detail": "Dataset not found"
}
```

---

### 4) Запуск анализа

- **Метод:** `POST`
- **Endpoint:** `/analyses`

**Описание:**
Создаёт новую аналитическую сессию для выбранного набора данных, формирует профиль весов и рассчитывает результаты ранжирования.

**Тело запроса:**
```json
{
  "dataset_id": 1,
  "normalization_method": "minmax",
  "clipping_enabled": true,
  "weight_method": "manual",
  "weights": {
    "import_share": 0.30,
    "import_growth": 0.20,
    "dependency_ratio": 0.20,
    "hhi_index": 0.15,
    "volatility": 0.15
  }
}
```

**Поля тела запроса:**
- `dataset_id` — идентификатор набора данных;
- `normalization_method` — способ нормализации;
- `clipping_enabled` — использовать ли clipping;
- `weight_method` — `manual` или `ahp`;
- `weights` — веса критериев.

**Пример ответа `201 Created`:**
```json
{
  "id": 1,
  "dataset_id": 1,
  "normalization_method": "minmax",
  "clipping_enabled": true,
  "weight_method": "manual",
  "status": "completed",
  "created_at": "2026-03-18T12:30:00",
  "weight_profile_id": 1,
  "result_count": 3
}
```

## 2. Реализация API

Ниже приведён пример учебной реализации API на FastAPI. Реализация покрывает требование по наличию методов GET, POST, PUT и DELETE, а также содержит более 6 endpoint-ов.

Файл с кодом: `lr4_api_fastapi.py`

```python
from datetime import datetime
from typing import Dict, List, Literal

from fastapi import FastAPI, HTTPException, Response, status
from pydantic import BaseModel, Field


app = FastAPI(title="Import Substitution Prioritization API", version="1.0.0")


class DatasetCreate(BaseModel):
    file_name: str
    source_format: Literal["csv", "xlsx"]
    records_count: int = Field(ge=0)
    status: str = "uploaded"


class DatasetResponse(DatasetCreate):
    id: int
    uploaded_at: datetime


class WeightProfileUpdate(BaseModel):
    import_share: float = Field(ge=0, le=1)
    import_growth: float = Field(ge=0, le=1)
    dependency_ratio: float = Field(ge=0, le=1)
    hhi_index: float = Field(ge=0, le=1)
    volatility: float = Field(ge=0, le=1)


class AnalysisCreate(BaseModel):
    dataset_id: int
    normalization_method: str
    clipping_enabled: bool
    weight_method: Literal["manual", "ahp"]
    weights: WeightProfileUpdate


datasets: Dict[int, dict] = {}
analyses: Dict[int, dict] = {}
weight_profiles: Dict[int, dict] = {}
results: Dict[int, List[dict]] = {}

dataset_seq = 1
analysis_seq = 1
profile_seq = 1


def next_dataset_id() -> int:
    global dataset_seq
    current = dataset_seq
    dataset_seq += 1
    return current


def next_analysis_id() -> int:
    global analysis_seq
    current = analysis_seq
    analysis_seq += 1
    return current


def next_profile_id() -> int:
    global profile_seq
    current = profile_seq
    profile_seq += 1
    return current


def generate_demo_results() -> List[dict]:
    return [
        {"tnved_code": "847130", "score": 0.81, "rank": 1},
        {"tnved_code": "851762", "score": 0.74, "rank": 2},
        {"tnved_code": "854231", "score": 0.69, "rank": 3},
    ]


@app.get("/api/v1/datasets")
def get_datasets():
    return list(datasets.values())


@app.post("/api/v1/datasets", status_code=status.HTTP_201_CREATED)
def create_dataset(payload: DatasetCreate):
    dataset_id = next_dataset_id()
    item = {
        "id": dataset_id,
        "file_name": payload.file_name,
        "source_format": payload.source_format,
        "records_count": payload.records_count,
        "uploaded_at": datetime.utcnow().isoformat(),
        "status": payload.status,
    }
    datasets[dataset_id] = item
    return item


@app.get("/api/v1/datasets/{dataset_id}")
def get_dataset(dataset_id: int):
    dataset = datasets.get(dataset_id)
    if not dataset:
        raise HTTPException(status_code=404, detail="Dataset not found")
    return dataset


@app.delete("/api/v1/datasets/{dataset_id}", status_code=status.HTTP_204_NO_CONTENT)
def delete_dataset(dataset_id: int):
    if dataset_id not in datasets:
        raise HTTPException(status_code=404, detail="Dataset not found")
    del datasets[dataset_id]
    return Response(status_code=status.HTTP_204_NO_CONTENT)


@app.post("/api/v1/analyses", status_code=status.HTTP_201_CREATED)
def create_analysis(payload: AnalysisCreate):
    if payload.dataset_id not in datasets:
        raise HTTPException(status_code=404, detail="Dataset not found")

    profile_id = next_profile_id()
    weight_profiles[profile_id] = {
        "id": profile_id,
        **payload.weights.model_dump(),
    }

    analysis_id = next_analysis_id()
    demo_results = generate_demo_results()

    analyses[analysis_id] = {
        "id": analysis_id,
        "dataset_id": payload.dataset_id,
        "normalization_method": payload.normalization_method,
        "clipping_enabled": payload.clipping_enabled,
        "weight_method": payload.weight_method,
        "status": "completed",
        "created_at": datetime.utcnow().isoformat(),
        "weight_profile_id": profile_id,
        "result_count": len(demo_results),
    }
    results[analysis_id] = demo_results
    return analyses[analysis_id]


@app.get("/api/v1/analyses/{analysis_id}/results")
def get_analysis_results(analysis_id: int):
    analysis = analyses.get(analysis_id)
    if not analysis:
        raise HTTPException(status_code=404, detail="Analysis not found")

    return {
        "analysis_id": analysis_id,
        "dataset_id": analysis["dataset_id"],
        "items": results.get(analysis_id, []),
    }


@app.put("/api/v1/weight-profiles/{profile_id}")
def update_weight_profile(profile_id: int, payload: WeightProfileUpdate):
    if profile_id not in weight_profiles:
        raise HTTPException(status_code=404, detail="Weight profile not found")

    updated = {"id": profile_id, **payload.model_dump()}
    weight_profiles[profile_id] = updated
    return updated
```
