# 2. Перечень функциональных требований

*   **FR-CMS-1:** Программа должна загружать статистику внешнеэкономической деятельности из открытых источников и сохранять её.
*   **FR-CMS-2:** Программа должна строить прогноз данных импорта на будущие периоды.
*   **FR-CMS-3:** Программа должна рассчитывать налоговые поступления от импорта по категориям товаров из ТН ВЭД.
*   **FR-CMS-4:** Программа должна рассчитывать налоговые поступления от локального производства товаров по категориям товаров из ТН ВЭД.
*   **FR-CMS-5:** Программа должна строить графики на основе рассчитанных данных.
*   **FR-CMS-6:** Программа должна показывать самые выгодные товары для импортозамещения.
*   **FR-CMS-7:** Программа должна предоставлять возможность авторизации.
*   **FR-CMS-8** Программа должна формировать отчеты с возможность экспортировать графики и таблицы в форматы PDF/Excel.

# 4. Перечень сделанных предположений

**Качество и доступность данных:**

Источники данных (ФТС России, ФНС, Росстат) предоставляют статистику в структурированном, машиночитаемом формате (CSV, XLS, API).

Данные являются достаточно полными, актуальными и достоверными для проведения значимого анализа.

Методология сбора и классификации данных (по ТН ВЭД) со временем не претерпевает кардинальных изменений, которые сделали бы невозможным сравнение данных за разные периоды.

**Модель прогнозирования:**

Для построения прогноза импорта достаточно исторических данных за прошедшие периоды. Система не учитывает форс-мажорные события, геополитические шоки или внезапные изменения законодательства.

Будет использоваться относительно простая, но эффективная модель прогнозирования, не требующая экстремальных вычислительных ресурсов.

**Методология расчета:**

Налоговые поступления от импорта и локального производства могут быть рассчитаны на основе аппроксимации (например, средней ставки пошлины или НДС по товарной группе) без необходимости подключения к внутренним системам ФНС.

Критерии "выгодности" для импортозамещения могут быть формализованы.

**Контекст использования:**

Пользователями системы являются аналитики, экономисты или госслужащие, имеющие базовое понимание экономической статистики и классификатора ТН ВЭД.

Система не подменяет собой процесс принятия решений, а является инструментом для анализа, предоставляющим рекомендации. Ответственность за окончательное решение лежит на человеке.

**Технический контекст:**

Система будет развернута в защищенном контуре организации (Минпромторга или фонда), что обеспечивает базовый уровень сетевой безопасности.

Для работы системы будет обеспечен стабильный доступ к интернету для загрузки актуальных данных из открытых источников.

# 5. Перечень нефункциональных требований

## 1. Производительность:
*   **NR-PERF-1:** Время отклика системы для любого пользовательского действия не должно превышать 2 секунд при средней нагрузке.

## 2. Надежность и доступность:
*   **NR-REL-1:** Система не должна аварийно завершать свою работу.

## 3. Безопасность:
*   **NR-SEC-1:** Логины и пароли для авторизации должны храниться в зашифрованном виде.
*   **NR-SEC-2:** Неавторизованный пользователь не должен иметь доступа к функционалу системы.

## 4. Удобство использования (Usability):
*   **NR-USA-1:** Интерфейс системы должен быть интуитивно понятным и требовать минимального обучения.

## 5. Сопровождаемость:
*   **NR-MAIN-1:** Система должна быть построена с использованием стандартных технологий и иметь полную техническую документацию для облегчения поддержки и дальнейшего развития.
